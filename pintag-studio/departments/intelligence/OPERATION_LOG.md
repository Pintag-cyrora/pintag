# Intelligence Department — Operation Log

*This is not a design document. It is an operational journal — evidence of real daily use, not a plan for future work. This log, together with `../_GRADUATION_CHECKLIST.md`, is what decides whether the Intelligence Department has actually earned Level 3 (Operated Daily), not whether its software exists.*

**The rule going forward: real usage creates the backlog.** Each day, instead of asking "what feature should we build?", ask "what slowed us down today while operating the department?" — that becomes the next improvement, and it gets recorded here, not invented in advance.

No entries are pre-filled. The first entry gets added the first day the department is actually operated for real (`npm run knowledge:review`, `npm run knowledge:dashboard`, retrieval used during real workflow work) — not the day the software was built. A fabricated entry would defeat the entire point of this log.
